Extract all information from this receipt/invoice image and return it as a JSON object.

  "date": "YYYY-MM-DD",
  "total_amount": 123.45,
  "currency": "INR",
  "subtotal": 111.11,

  "receipt_number": "Invoice number if available",
  "items": [
    {

      "total": 75.00,
      "tax_rate_percent": 18.0,
      "tax_amount": 11.44,
      "hsn_sac_code": "HSN or SAC code if printed"
    }
  ],
  "taxes": [
    {
      "name": "Tax label as printed (e.g. CGST, SGST, IGST, VAT, GST, Service Tax)",
      "rate_percent": 9.0,

    }

INSTRUCTIONS:
1. Extract ALL line items with their quantities, prices, and totals
2. Use exact values from the receipt
3. If quantity is not shown, use 1.0
4. If unit_price is not shown, set it equal to total
5. Preserve item names exactly as written
6. Include ALL items, even if there are many
7. Return ONLY valid JSON, no additional text

TAX INSTRUCTIONS:
8. Populate the "taxes" array with every tax line printed on the receipt (CGST, SGST, IGST, VAT, GST, Service Tax, Cess, etc.)
9. Set "tax_total" to the sum of all tax amounts
10. If per-item tax details (rate, amount, HSN/SAC) are printed, include them on each item; otherwise omit those item fields
11. If no tax information is present anywhere on the receipt, set "taxes" to [] and "tax_total" to 0

LOYALTY / MEMBERSHIP INSTRUCTIONS:
12. If any loyalty, rewards, membership, or points section appears, populate the "loyalty" object
13. Capture points earned on this transaction, points redeemed, and running balance if shown
14. If the program shows monetary wallet values (cashback, store credit) instead of or in addition to points, include those in the monetary_value_* fields
15. If no loyalty information is present, omit the "loyalty" key entirely
16. If a field within "loyalty" is not shown on the receipt, omit that key

For grocery receipts, extract every single product line item.
